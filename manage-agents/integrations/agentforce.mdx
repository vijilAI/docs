You can register Agentforce-based agents on Vijil.

## Prerequisites

- You have access to Agentforce and at least one configured Agentforce agent.
- You have any credentials or URLs required to invoke your Agentforce agent through its API.
- You have access to a Vijil team and permission to manage **[Keys](https://evaluate.vijil.ai/keys)** and **[Agents](/tutorials/manage-agents/)**.

## Store Agentforce Credentials in Vijil

1. In Vijil, go to **[Keys](https://evaluate.vijil.ai/keys) > Add new key**.
2. From the **Model Hub** dropdown, select **Agentforce**.
3. Enter your Agentforce credentials or token and any required configuration fields (such as the Agentforce endpoint URL) in the hub configuration.
4. Save the key configuration.

Once saved, this key can be reused across multiple Agentforce-based agents.

## Register an Agent on Vijil

With an Agentforce key configured, you can register an agent that uses Agentforce:

![Setting up an Agentforce-based Agent in Vjil](/images/manage-agents/integrations/agentforce/agentforce.png)

1. Press **Register Agent**.
2. Enter an [Agent](/tutorials/manage-agents/) name.
3. Select **Agentforce** as the **Hub**.
4. Enter an **Agent ID**
5. Select an existing Agentforce API key you configured earlier, or create a new one.
6. Add a **System Prompt** if you want Vijil to send a default system prompt with every request to this agent.

## Enterprise

If your organization uses enterprise-only features with Vijil:

1. Add an **MCP server URL** so the agent can access MCP tools exposed by that server.
2. Add an **A2A Agent Card URL** so this agent can participate in agent-to-agent workflows defined in your enterprise.
