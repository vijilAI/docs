---
title: "SEO and Image Guidelines for Vijil Docs"
description: "Best practices for authoring SEO-friendly content and optimized images in the Vijil documentation."
---

Keeping Vijil docs fast and discoverable requires a few consistent practices around page metadata and images.

## Page metadata

- Always set a clear `title` and concise `description` in the frontmatter of new pages.
- Write descriptions for humans first — summarize the page in 1–2 sentences using the key terms users would search for.
- Avoid duplicating the exact same `description` across many pages unless they are intentional variants of the same content.

Example:

```mdx
---
title: "Configure Dome guardrails"
description: "Learn how to configure Vijil Dome guardrails to protect agents in production from unsafe or insecure behavior."
---
```

## Image usage

- Prefer static screenshots and diagrams over animated GIFs. If you must use GIFs, keep their dimensions and file size as small as practical.
- Use descriptive alt text that explains what the user should notice in the image.
- Place large images below the introductory paragraph so they are not the first thing rendered above the fold.

### Markdown images

When adding images with Markdown syntax:

- Use a descriptive alt attribute.
- Keep filenames lowercase and hyphenated, e.g. `/images/quickstart-agent-registry.png`.

```mdx
![Agent Registry showing registered agents with status and Trust Score](/images/quickstart-agent-registry.png)
```

### HTML `<img />` tags

For HTML `<img />` tags in MDX:

- Add explicit `width` and `height` when known to improve layout stability.
- Use `loading="lazy"` for images that are not immediately visible when the page loads.

```mdx
<img
  src="/images/agent-registry.png"
  alt="Agent Registry showing registered agents with status, access level, and Trust Score"
  width={1280}
  height={720}
  loading="lazy"
/>
```

## Choosing formats

- Prefer modern formats like WebP for new assets when available.
- For existing PNGs and JPEGs, keep the original path but consider adding a WebP version alongside it as `/images/...-webp`.
- Avoid embedding very large images directly in long lists or tables; summarize with text and link to a focused section if needed.
