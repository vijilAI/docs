Agents often stall before production: security and compliance need **proof**, not demos. Vijil turns that into **measurable trust**, systematic evaluation, runtime protection, and a feedback loop from real behavior.

[**Trust Score**](/concepts/trust-score/introduction) summarizes how an agent performs across [**reliability**](/concepts/trust-score/reliability), [**security**](/concepts/trust-score/security), and [**safety**](/concepts/trust-score/safety); so reviews can reference evidence instead of opinions. Under that umbrella, **Diamond** probes behavior in depth, while **Dome** enforces [**Guardrails**](/concepts/defense/guardrail) at runtime.

| Dimension | Question | Learn more |

## Build, Ship, Run, Evolve

Agent prototypes are everywhere—but most do not make it to production. To scale AI, enterprises face a critical choice: ship fast or ship resilient.

- Agent prototypes are everywhere—but most do not make it to production
- Shipping fast increases failure rates and elevates risk; without objective trust evidence, agents are in security limbo
- Generic red-teaming and [Guardrail](/concepts/defense/guardrail) do not ensure agents are safe for their specific context
- Failure insights do not feed back into development, leaving the trust gap open

# **Getting Started with MCP in Vijil (Hands-On Guide)**

If you’re new to **MCP**, don’t worry, because you don’t need to understand every detail to start using it.

We created a guide that focuses on the practical side: 

- how to get **MCP** working with **Vijil** and how it actually helps you when building or testing AI agents.

**MCP (Model Context Protocol)** is basically a standard way for AI agents and tools to communicate with each other. Instead of building custom integrations every time, **MCP** gives you a shared structure for exposing tools, APIs, and workflows to an AI system. 

In the Vijil ecosystem, **MCP** is used to connect AI assistants and developer tools with the **Vijil** Console and APIs. ([<u>PyPI</u>](https://pypi.org/project/vijil-console-mcp/?utm_source=chatgpt.com))

# **What You’re Actually Setting Up**

When you configure **MCP** with **Vijil**, you’re basically making a connection between:

- your AI assistant (like Claude Code)
- and the Vijil platform

Once they are successfully connected, the assistant can directly interact with Vijil through tools and commands.

That means you can do things like:

- list agents
- launch evaluations
- review Trust Scores
- Check evaluation results
- Create red-team campaigns

and all that without manually clicking through the Console every time. ([<u>PyPI</u>](https://pypi.org/project/vijil-console-mcp/?utm_source=chatgpt.com))

\

**Headsup Before You Start**

You’ll need a few things first:

- Python 3.10\+
- A Vijil account with API access
- Claude Code or another MCP-compatible client
- Access to your Vijil Console environment ([<u>PyPI</u>](https://pypi.org/project/vijil-console-mcp/?utm_source=chatgpt.com))

To get started, follow this simple 5-step guide. 

# **Step 1: Install the MCP Server**

The easiest way to get started is through pip.

pip install vijil-console-mcp

This installs:

- The MCP server
- and the Vijil CLI are automatically installed as a dependency. ([<u>PyPI</u>](https://pypi.org/project/vijil-console-mcp/?utm_source=chatgpt.com))

If you like the isolated installs more, you can also use:

pipx install vijil-console-mcp

\

# **Step 2: Connect to Your Vijil Environment**

Now you need to tell the CLI where your Vijil Console API lives.

Run: vijil init --url [https://console-api.example.com](https://console-api.example.com)

If you don’t include the URL, Vijil will prompt you interactively.

Here is an example:

vijil init

\

The CLI verifies the connection before saving the configuration, which helps catch setup mistakes early. ([<u>PyPI</u>](https://pypi.org/project/vijil-console-mcp/?utm_source=chatgpt.com))

# **Step 3: Authenticate**

Next, you should log in to your Vijil account:

vijil login

\

You’ll be prompted for:

- email
- password

If your account belongs to multiple teams, select the active one:

vijil team list

vijil team use \<team\_id\>

\

At this point, your local environment is connected to Vijil and ready to provide tools through MCP. ([<u>PyP</u>](https://pypi.org/project/vijil-console-mcp/?utm_source=chatgpt.com))

# **Step 4: Add MCP Configuration**

Now create a file called:

.mcp.json

\

Place it in your project root.

Add this configuration:

{

  "mcpServers": {

    "vijil": {

      "type": "stdio",

      "command": "vijil-mcp"

    }

  }

}

\

This tells your AI assistant:

“Whenever you need Vijil tools, start the vijil-mcp process locally.”

And that’s it. There’s no need for complex networking or server deployment. ([<u>PyPI</u>](https://pypi.org/project/vijil-console-mcp/?utm_source=chatgpt.com))

\

# **Step 5: Start Using Vijil Tools**

Open Claude Code (or your MCP-compatible client) inside the project directory.

If everything is configured correctly, the Vijil tools should appear automatically.

You can now ask things like:

- List my agents
- Run a safety evaluation on my support agent
- Show the latest Trust Score
- Create a red-team campaign

\

How does this look behind the scene:

1. your assistant calls the MCP server
2. the MCP server runs Vijil CLI commands
3. the CLI communicates with the Vijil Console API
4. results are returned back to the assistant ([<u>PyPI</u>](https://pypi.org/project/vijil-console-mcp/?utm_source=chatgpt.com))

The nice part is that you don’t have to set all of this up manually. 

# **Understanding the Flow**

A lot of developers get confused in all of this because “MCP server” sounds like some hosted backend service.

In this setup, it’s actually much simpler, and at the same time, we did our best to explain everything in detail.

The **MCP** server runs locally on your machine, and your assistant starts it automatically when needed. ([<u>PyPI</u>](https://pypi.org/project/vijil-console-mcp/?utm_source=chatgpt.com))

The flow looks like this:

Claude Code

     ↓

vijil-mcp

     ↓

Vijil CLI

     ↓

Vijil Console API

\

So when you ask:

“Run an evaluation”

Your assistant is really triggering Vijil commands through that chain.

# **Common Problems (and Easy Fixes)**

## **“vijil CLI not found”**

Install the CLI again:

pip install vijil-console

\

## **“Vijil CLI not configured”**

Run initialization again:

vijil init

\

## **“Session expired”**

Log in again:

vijil login

\

## **Tools don’t appear in Claude Code**

Usually this means:

- .mcp.json is missing
- or Claude Code needs to be restarted

Make sure the file is in the project root. ([<u>PyPI</u>](https://pypi.org/project/vijil-console-mcp/?utm_source=chatgpt.com))

# **Why is this Important?**

Without **MCP**, developers often have to build custom integrations between assistants and internal tools.

With **MCP**, tool integration becomes easier because there’s a standardized way to expose and use tools. 

For Vijil specifically, it means you can:

- interact with evaluations faster
- automate trust workflows
- Access agent operations directly from your AI assistant
- Reduce manual work that repeats itself

With **MCP**, there’s less need to switch between dashboards, terminals, and APIs since the assistant becomes part of the workflow.
