---
title: 'Custom Harnesses'
description: 'Create targeted Evaluations with custom Harnesses scoped to your Agent, Personas, and Policies.'
---

Custom Harnesses let you generate test [Probes](/concepts/evaluation-components/probe) tailored to your specific [Agent](/tutorials/manage-agents), compliance [Policies](/owner-guide/simulate-environment/policies), and user [Personas](/owner-guide/simulate-environment/personas). Vijil uses your configuration to generate adversarial Probes in the background, which you can then use in evaluations alongside or instead of standard Harnesses.

## List Harnesses

List standard Harnesses to see what is available before creating a custom one.

<CodeGroup>
```bash title="CLI"
vijil harness list
```

```bash title="API"
curl -s "$VIJIL_URL/harnesses/?team_id=$TEAM_ID" \
  -H "Authorization: Bearer $TOKEN"
```
</CodeGroup>

Standard Harnesses include [Reliability](/concepts/trust-score/reliability), [Safety](/concepts/trust-score/safety), and [Security](/concepts/trust-score/security).

To list your existing custom Harnesses:

<CodeGroup>
```bash title="CLI"
vijil harness custom-list
vijil harness custom-list --agent-id "$AGENT_ID" --status active
```

```bash title="API"
curl -s "$VIJIL_URL/custom-harnesses/?team_id=$TEAM_ID" \
  -H "Authorization: Bearer $TOKEN"
```
</CodeGroup>

## Create a Custom Harness

Harness creation is asynchronous, so Vijil generates Probes in the background based on your [Agent](/tutorials/manage-agents), [Personas](/owner-guide/simulate-environment/personas), and [Policies](/owner-guide/simulate-environment/policies). The API and CLI return a Harness ID immediately; poll for status until it becomes `active`.

<Tabs>
  <Tab title="CLI">

  ```bash
  vijil harness custom-create \
    --name "Customer Support Harness" \
    --agent-id "$AGENT_ID"
  ```

  Add [Personas](/owner-guide/simulate-environment/personas) and [Policies](/owner-guide/simulate-environment/policies) to generate more targeted Probes:

  ```bash
  vijil harness custom-create \
    --name "Customer Support Harness" \
    --agent-id "$AGENT_ID" \
    --persona-ids '["<persona_id>"]' \
    --policy-ids '["<policy_id>"]' \
    --system-prompt "You are a customer support agent for Acme Corp."
  ```

  | Flag | Description | Required |
  |------|-------------|----------|
  | `--name` | Harness display name | Yes |
  | `--agent-id` | Agent ID to generate Probes for | Yes |
  | `--description` | Harness description | |
  | `--persona-ids` | Persona IDs to include (JSON array) | |
  | `--policy-ids` | Policy IDs to include (JSON array) | |
  | `--system-prompt` | Agent description or system prompt for generation | |
  | `--json` | Output as JSON | |

  </Tab>
  <Tab title="API">

  ```bash
  curl -s -X POST "$VIJIL_URL/custom-harnesses/" \
    -H "Authorization: Bearer $TOKEN" \
    -H "Content-Type: application/json" \
    -d "{
      \"name\": \"Customer Support Harness\",
      \"agent_id\": \"$AGENT_ID\",
      \"persona_ids\": [\"<persona_id>\"],
      \"policy_ids\": [\"<policy_id>\"],
      \"system_prompt\": \"You are a customer support agent for Acme Corp.\"
    }"
  ```

  Response:

  ```json
  {
    "id": "h1i2j3k4-...",
    "name": "Customer Support Harness",
    "status": "draft",
    "agent_id": "a1b2c3d4-...",
    "created_at": 1712505600
  }
  ```

  Save the Harness ID:

  ```bash
  export HARNESS_ID="h1i2j3k4-..."
  ```

  | Field | Description | Required |
  |-------|-------------|----------|
  | `name` | Harness display name (max 255 chars) | Yes |
  | `agent_id` | UUID of the Agent to generate Probes for | Yes |
  | `description` | Harness description | |
  | `persona_ids` | Persona IDs to include | |
  | `policy_ids` | Policy IDs to include | |
  | `system_prompt` | Agent description or system prompt for generation | |

  </Tab>
  <Tab title="MCP">

  With the [Vijil MCP server](/developer-guide/agentic/mcp) configured, ask Claude Code:

  <Prompt description="Create a custom Harness called 'Customer Support Harness' for agent a1b2c3d4-...">
  Create a custom Harness called 'Customer Support Harness' for agent a1b2c3d4-...
  </Prompt>

  To include [Personas](/owner-guide/simulate-environment/personas) and [Policies](/owner-guide/simulate-environment/policies):

  <Prompt description="Create a custom Harness called 'Customer Support Harness' for agent a1b2c3d4-... using persona p1q2r3s4-... and policy p5q6r7s8-...">
  Create a custom Harness called 'Customer Support Harness' for agent a1b2c3d4-... using persona p1q2r3s4-... and policy p5q6r7s8-...
  </Prompt>

  </Tab>
</Tabs>

## Check Status

Harness generation status progresses from `draft` to `active`. It may also become `failed`.

<CodeGroup>
```bash title="CLI"
vijil harness custom-get <harness_id>
```

```bash title="API"
curl -s "$VIJIL_URL/custom-harnesses/$HARNESS_ID" \
  -H "Authorization: Bearer $TOKEN"
```
</CodeGroup>

## View Generated Probes

Once the Harness status is `active`, retrieve the generated Probes:

<CodeGroup>
```bash title="CLI"
vijil harness custom-prompts <harness_id> --json
```

```bash title="API"
curl -s "$VIJIL_URL/custom-harnesses/$HARNESS_ID/prompts" \
  -H "Authorization: Bearer $TOKEN"
```
</CodeGroup>

## Use in an Evaluation

Pass the Harness name with `--harness-type custom` when running an evaluation:

<CodeGroup>
```bash title="CLI"
vijil eval run \
  --agent-id "$AGENT_ID" \
  --harness-names '["Customer Support Harness"]' \
  --harness-type custom \
  --wait
```

```bash title="API"
curl -s -X POST "$VIJIL_URL/evaluations/" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{
    \"agent_id\": \"$AGENT_ID\",
    \"team_id\": \"$TEAM_ID\",
    \"harness_names\": [\"Customer Support Harness\"],
    \"harness_type\": \"custom\"
  }"
```
</CodeGroup>

## Cancel and Delete

<CodeGroup>
```bash title="Cancel"
vijil harness custom-cancel <harness_id>
```

```bash title="Delete"
vijil harness custom-delete <harness_id>
vijil harness custom-delete <harness_id> --yes
```
</CodeGroup>

Use `POST /custom-harnesses/{harness_id}/cancel` and `DELETE /custom-harnesses/{harness_id}` for the REST API equivalents.
