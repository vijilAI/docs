Vijil can evaluate agents and models hosted on supported model providers. Use this guide to identify the provider credentials, endpoint details, and rate-limit settings you need before running an evaluation.

## Supported Providers

This table is a provider reference, not a static model allowlist. For providers with model catalogs, choose a model that your provider account can access and that Vijil supports for that provider.

| Provider | What Vijil can evaluate | Model catalog |
|----------|-------------------------|---------------|
| OpenAI | OpenAI-hosted models available to your account | [OpenAI models](https://platform.openai.com/docs/models) |
| Anthropic | Claude models available to your Anthropic account | [Anthropic models](https://docs.anthropic.com/en/docs/about-claude/models/overview) |
| AWS Bedrock | Bedrock foundation models and Bedrock Agents configured in AWS | [AWS Bedrock models](https://docs.aws.amazon.com/bedrock/latest/userguide/models-supported.html) |
| Google Vertex AI | Gemini and partner models available through Vertex AI | [Google models on Vertex AI](https://cloud.google.com/vertex-ai/generative-ai/docs/learn/models) |
| DigitalOcean | DigitalOcean AI Platform agents and endpoints | [DigitalOcean AI Platform docs](https://docs.digitalocean.com/products/ai-platform/) |
| Custom endpoints | OpenAI-compatible endpoints configured by your team | Use your endpoint's model documentation |

## Provider Setup

Store provider credentials in Vijil before running an evaluation. Use the exact model identifier, deployment name, endpoint, or agent identifier expected by the provider and your Vijil configuration.

### API-based Providers

For OpenAI and Anthropic, store the provider API key and use a model identifier from the provider catalog. Your provider account must have access to the model you select.

### AWS Bedrock

For Bedrock foundation models, store AWS credentials with permission to invoke the target model and specify the AWS region. Use the model ID format from the AWS model catalog for your selected region, and confirm any Vijil-specific formatting requirements before publishing examples.

For Bedrock Agents, provide the Bedrock agent identifier, agent alias identifier, region, and AWS access configuration.

### Google Vertex AI

Authenticate with Google Cloud using credentials that can call Vertex AI generative models. Use model identifiers from the Google model catalog and confirm that the selected model is available in your project and region.

### DigitalOcean

Use the DigitalOcean AI Platform endpoint and agent credentials for the agent or endpoint you want Vijil to evaluate.

### Custom Endpoints

Evaluate any compatible endpoint by storing its API key and providing the endpoint URL. Use the model name and request shape that your endpoint expects.

## Rate Limiting

Control evaluation pace to avoid provider throttling and to avoid overwhelming the agent under test. Do not rely on a single docs-wide requests-per-minute value. Set the Vijil-side limit below the strictest limit from your provider account, selected model, cloud project or region, and application capacity.

| Provider | Where to verify limits |
|----------|------------------------|
| OpenAI | [OpenAI rate limits](https://platform.openai.com/docs/guides/rate-limits) |
| Anthropic | [Anthropic rate limits](https://docs.anthropic.com/en/api/rate-limits) |
| AWS Bedrock | [Amazon Bedrock quotas](https://docs.aws.amazon.com/bedrock/latest/userguide/quotas.html) |
| Google Vertex AI | [Vertex AI quotas](https://cloud.google.com/vertex-ai/generative-ai/docs/quotas) |
| DigitalOcean | DigitalOcean account settings and endpoint-specific limits |
| Custom endpoints | Endpoint-specific limits set by your service |

<Tip>
Start with a conservative request limit for new evaluations, then increase it only after confirming that provider dashboards and Vijil evaluation logs show enough headroom.
</Tip>
